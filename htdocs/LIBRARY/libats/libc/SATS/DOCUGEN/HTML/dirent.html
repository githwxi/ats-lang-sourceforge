ATSLIB/libc/dirent

The functions declared in this package are mostly the API for the corresponding libc-functions declared in [dirent.h], and many more details on these functions can often be found in the Linux Programmer's manual. See dirent.sats for the SATS file in ATSLIB where these functions are declared.

    readdir

    Synopsis

    fun
readdir
(
  dirp: !DIRptr1
) :<!refwrt> [l:addr]
(
  option_v(vtakeout0 (dirent@l), l > null) | ptr (l)
) = "mac#%" // end of [readdir]

    Description

    This function returns a pointer to a dirent structure representing the next directory entry in the directory stream pointed to by [dirp]. It returns NULL on reaching the end of the directory stream or if an error is encountered.

    readdir_r

    Synopsis

    fun
readdir_r
(
  dirp: !DIRptr1
, ent0: &dirent? >> opt (dirent, l > null)
, result: &ptr? >> ptr(l)
) :<!wrt> #[l:addr;i:int | i >= 0] int(i) = "mac#%"

    Description

    This function is a reentrant version of readdir. It reads the next directory entry from the directory stream [dirp], and returns it in the caller-allocated buffer pointed to by [entry]. Also, it places a pointer to the returned item in [result]. If the end of the directory stream was encountered, then NULL is instead placed in [result].
